Anomalib Studio is a low/no-code web application that lets you train and deploy anomaly detection models through a visual interface. It is designed for operational environments where you need to connect cameras, train a model, and route results to industrial systems — without writing code.

The source code lives in the [`application/`](https://github.com/open-edge-platform/anomalib/tree/main/application) directory of the Anomalib repository.

## Key capabilities

- **Flexible input sources** — Connect USB cameras, IP cameras, or use a folder of images as the training and inference input.
- **Visual training workflow** — Configure and run model training through the web UI without touching the command line.
- **Industrial output** — Send inference results directly to downstream systems via ROS messages, MQTT, and other protocols.
- **Model management** — Track trained models and switch between them from the interface.

## Architecture

Anomalib Studio is built as a two-tier web application:

- **Frontend** — A React application served on port 3000 in development or bundled into the backend for production.
- **Backend** — A Python [FastAPI](https://fastapi.tiangolo.com/) server that manages training, inference, camera streams, and database state. It uses [OpenVINO](https://docs.openvino.ai/) for optimized model inference.
