Anomalib Studio can be run in two ways: as a Docker container for a self-contained deployment, or as separate frontend and backend processes for development. Choose the approach that fits your use case.

<Note>
  Clone the Anomalib repository before following either setup path:

  ```bash
  git clone https://github.com/open-edge-platform/anomalib.git
  ```
</Note>

<Tabs>
  <Tab title="Docker">

## Prerequisites

- [Docker](https://docs.docker.com/get-docker/) and [Docker Compose](https://docs.docker.com/compose/install/) — Docker Compose v2 is recommended (`docker compose`, not `docker-compose`)
- **Intel XPU builds only:** Intel GPU drivers and Intel oneAPI runtime installed on the host
- **CUDA builds only:** [NVIDIA Container Toolkit](https://docs.nvidia.com/datacenter/cloud-native/container-toolkit/install-guide.html) configured so Docker can access the GPU

## Setup

<Steps>
  <Step title="Navigate to the Docker directory">
    ```bash
    cd anomalib/application/docker
    ```

    All subsequent Docker commands should be run from this directory.
  </Step>
  <Step title="Build and start the container">
    Choose the build target that matches your hardware.

    <Tabs>
      <Tab title="CPU">
        Use this if you don't have a GPU or want a hardware-agnostic setup. This is the default and works on any machine with Docker installed.

        ```bash
        docker compose build
        docker compose up
        ```
      </Tab>
      <Tab title="Intel XPU">
        Use this for Intel discrete or integrated GPUs. Requires Intel GPU drivers and the oneAPI runtime on the host.

        ```bash
        AI_DEVICE=xpu docker compose build
        AI_DEVICE=xpu docker compose up
        ```
      </Tab>
      <Tab title="NVIDIA CUDA">
        Use this for NVIDIA GPUs. Uses a separate Compose file configured for the CUDA runtime.

        ```bash
        AI_DEVICE=gpu docker compose -f docker-compose.cuda.yaml build
        AI_DEVICE=gpu docker compose -f docker-compose.cuda.yaml up
        ```
      </Tab>
    </Tabs>

    <Note>
      To run the containers in the background without occupying the terminal, append `-d` to the `docker compose up` command.
    </Note>
  </Step>
  <Step title="Open the application">
    Once the containers are running, navigate to:

    ```
    http://localhost:8000
    ```

    Allow a few seconds for all services to initialize before the UI becomes available.
  </Step>
</Steps>

## Stopping the application

```bash
docker compose down
```

## Troubleshooting

If the application does not start or the UI is unreachable, check the container logs:

```bash
docker compose logs -f
```

Look for lines indicating which port the service is listening on, or any startup failures related to missing drivers or permissions.

## Prerequisites

- [uv](https://docs.astral.sh/uv/getting-started/installation/) — Python package manager used for the backend
- [Node.js](https://nodejs.org/) — Required to install and run the frontend
- Python 3.13 or later

## Setup

<Steps>
  <Step title="Install backend dependencies">
    Navigate to the backend directory and install dependencies using `uv`. Choose the extra that matches your hardware.

    <Tabs>
      <Tab title="Intel XPU">
        ```bash
        cd anomalib/application/backend
        uv sync --extra xpu
        ```
      </Tab>
      <Tab title="NVIDIA CUDA">
        ```bash
        cd anomalib/application/backend
        uv sync --extra cu130
        ```
      </Tab>
      <Tab title="CPU">
        ```bash
        cd anomalib/application/backend
        uv sync --extra cpu
        ```
      </Tab>
    </Tabs>
  </Step>
  <Step title="Install frontend dependencies">
    In a separate terminal, navigate to the UI directory and install Node.js dependencies:

    ```bash
    cd anomalib/application/ui
    npm install
    ```
  </Step>
  <Step title="Start the backend server">
    From the backend directory, run:

    ```bash
    cd anomalib/application/backend
    ./run.sh
    ```

    The FastAPI server will start and listen for requests from the frontend.

    <Note>
      To seed the database with example data before starting, set `SEED_DB=true`:

      ```bash
      SEED_DB=true ./run.sh
      ```
    </Note>
  </Step>
  <Step title="Start the frontend server">
    From the UI directory, run:

    ```bash
    cd anomalib/application/ui
    npm run start
    ```
  </Step>
  <Step title="Open the application">
    Navigate to [http://localhost:3000](http://localhost:3000) in your browser to use Anomalib Studio.
  </Step>
</Steps>

## Frontend commands

| Command | Description |
|---|---|
| `npm run start` | Start the development server |
| `npm run build` | Build the app for production |
| `npm run preview` | Preview the production build locally |
| `npm run lint` | Run the linter |
| `npm run type-check` | Run TypeScript type checking |
| `npm run test:unit` | Run unit tests |
| `npm run test:component` | Run component tests |
