Anomalib provides two models for video anomaly detection. Both models operate on video clips (sequences of frames) and use spatio-temporal feature representations that are not present in image models.

<CardGroup cols={2}>

## Video-specific concepts

### Frame-level vs clip-level anomaly detection

Video models process **clips** — short sequences of consecutive frames — rather than individual images. This allows them to model temporal dynamics such as motion, pose changes, and object trajectories.

- **Frame-level scoring**: Assign an anomaly score to each individual frame within the clip.
- **Clip-level scoring**: Assign a single anomaly score to the entire clip.

Both AI-VAD and FUVAS produce frame-level anomaly scores.

### Video datamodule configuration

Video datamodules expose clip-level parameters that control how frames are sampled:

```python
from anomalib.data import Avenue
from anomalib.data.utils import VideoTargetFrame

datamodule = Avenue(
    clip_length_in_frames=2,   # number of frames per clip
    frames_between_clips=1,    # stride between consecutive clips
    target_frame=VideoTargetFrame.LAST,  # which frame to score
)
```

`VideoTargetFrame` controls which frame in the clip receives the anomaly score:

| Value | Behavior |
|-------|----------|
| `FIRST` | Score the first frame of the clip |
| `LAST` | Score the last frame of the clip |
| `MID` | Score the middle frame |
| `ALL` | Score all frames |

## AI-VAD

AI-VAD (Attribute-based Representations for Accurate and Interpretable Video Anomaly Detection) extracts regions of interest from video frames using a combination of object detection (Faster R-CNN) and foreground detection based on inter-frame pixel differences. For each region, it computes three types of attribute features:

- **Velocity features** — histogram of optical flow magnitudes within the region
- **Pose features** — skeleton keypoint coordinates from a pose estimator
- **Deep features** — deep CNN embeddings of the region crop

Each feature type is modeled by a separate density estimator (GMM for velocity, KNN for pose and deep features). Anomaly scores are derived from the density of these features under the fitted models.

**Learning type:** ONE_CLASS  
**Paper:** [AI-VAD: Attribute-based Representations for Accurate and Interpretable Video Anomaly Detection](https://arxiv.org/abs/2212.00789)

### Parameters

| Parameter | Default | Description |
|-----------|---------|-------------|
| `box_score_thresh` | `0.7` | Confidence threshold for bounding box predictions |
| `persons_only` | `False` | Only include regions labeled as persons |
| `min_bbox_area` | `100` | Minimum bounding box area in pixels |
| `max_bbox_overlap` | `0.65` | Maximum allowed overlap between bounding boxes |
| `enable_foreground_detections` | `True` | Add foreground detections from frame differences |
| `foreground_kernel_size` | `3` | Gaussian kernel size for foreground detection |
| `foreground_binary_threshold` | `18` | Pixel difference threshold for foreground detection (0–255) |
| `n_velocity_bins` | `1` | Number of bins in velocity histogram |
| `use_velocity_features` | `True` | Include velocity features |
| `use_pose_features` | `True` | Include pose features |
| `use_deep_features` | `True` | Include deep CNN features |
| `n_components_velocity` | `2` | GMM components for velocity density estimation |
| `n_neighbors_pose` | `1` | KNN neighbors for pose density estimation |
| `n_neighbors_deep` | `1` | KNN neighbors for deep feature density estimation |

### Usage

```python
from anomalib.data import Avenue
from anomalib.data.utils import VideoTargetFrame
from anomalib.models import AiVad
from anomalib.engine import Engine

# Configure the Avenue dataset for video anomaly detection
datamodule = Avenue(
    clip_length_in_frames=2,
    frames_between_clips=1,
    target_frame=VideoTargetFrame.LAST,
)

# Initialize AI-VAD with default parameters
model = AiVad()

# Train and evaluate
engine = Engine()
engine.fit(model=model, datamodule=datamodule)
predictions = engine.predict(model=model, datamodule=datamodule)
```

To configure which features AI-VAD uses:

```python
# Use only deep features (disable velocity and pose)
model = AiVad(
    use_velocity_features=False,
    use_pose_features=False,
    use_deep_features=True,
)

# Only detect person regions with higher confidence threshold
model = AiVad(
    persons_only=True,
    box_score_thresh=0.8,
)
```

<Tip>
  AI-VAD is interpretable — you can identify which attribute (velocity, pose, or deep features) contributed most to a region's anomaly score, making it useful for understanding what type of anomaly was detected.
</Tip>

## FUVAS

FUVAS (Few-shot Unsupervised Video Anomaly Segmentation via Low-Rank Factorization of Spatio-Temporal Features) extracts spatio-temporal features from video clips using a pretrained 3D CNN or transformer backbone (default: X3D). During training, a PCA transformation is fitted to the embeddings from normal clips. At inference, anomaly scores are based on the feature reconstruction error after projecting through the fitted PCA model.

**Learning type:** ONE_CLASS  
**Paper:** [FUVAS: Few-shot Unsupervised Video Anomaly Segmentation via Low-Rank Factorization of Spatio-Temporal Features](https://ieeexplore.ieee.org/abstract/document/10887597) (ICASSP 2025)

### Parameters

| Parameter | Default | Description |
|-----------|---------|-------------|
| `backbone` | `"x3d_s"` | Pretrained 3D CNN or transformer backbone |
| `layer` | `"blocks.4"` | Backbone layer to extract features from |
| `pre_trained` | `True` | Use pretrained backbone weights |
| `spatial_pool` | `True` | Apply spatial pooling on extracted features |
| `pooling_kernel_size` | `1` | Kernel size for spatial pooling |
| `pca_level` | `0.98` | Variance ratio to preserve in PCA (between 0 and 1) |

### Usage

```python
from anomalib.data import UCSDped
from anomalib.models import Fuvas
from anomalib.engine import Engine

# Load UCSD Pedestrian dataset
datamodule = UCSDped()

# Initialize FUVAS with default parameters
model = Fuvas()

# Train and evaluate
engine = Engine()
engine.fit(model=model, datamodule=datamodule)
predictions = engine.predict(model=model, datamodule=datamodule)
```

To customize the backbone and PCA settings:

```python
model = Fuvas(
    backbone="x3d_s",
    layer="blocks.4",
    pca_level=0.95,   # retain 95% of variance
    spatial_pool=True,
)
```

### Example results on UCSDped

| Metric | Value |
|--------|-------|
| `frame_AUROC` | 0.914 |
| `mean_F1Score` | 0.935 |
| `pixel_AUROC` | 0.976 |

<Note>
  FUVAS does not perform gradient updates during training. The backbone remains frozen and only the PCA transformation is fitted to the extracted embeddings.
</Note>

## Choosing a video model

| Consideration | AI-VAD | FUVAS |
|---------------|--------|-------|
| Interpretability | High — attribute scores per region | Low — PCA reconstruction error |
| Setup complexity | Higher — requires object detector and pose estimator | Lower — single backbone |
| Few-shot data | No — requires full normal training set | Yes — designed for few-shot settings |
| Best for | Pedestrian/person anomalies | General video anomaly segmentation |
| Backbone | Faster R-CNN + pose estimator | 3D CNN (X3D) |
